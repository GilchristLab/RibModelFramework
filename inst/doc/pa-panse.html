AnaCoDa: PA and PANSE Models for Ribosome Footprinting Data

Cedric Landerer, Alexander L. Cope & Michael A. Gilchrist

2026-05-12

AnaCoDa provides two models for analyzing ribosome footprinting (RFP) data: the PA (PAusing time) model and its extension PANSE (PAusing time plus NonSense Error). Both models use a negative binomial likelihood to describe per-codon RFP counts, estimating pausing shape (\(\alpha\)), elongation rate (\(\lambda'\)), and gene-level expression (\(\psi\)) for each codon and gene in the genome.

PANSE additionally estimates per-codon nonsense error (NSE) rates, which reflect the probability that a translating ribosome terminates prematurely at a given codon. These NSE rates modulate the effective expression seen at downstream positions.

Input data format

PA and PANSE models require ribosome footprinting count data rather than coding sequences. The input file is a comma-separated table with one row per gene-codon combination:

GeneID,count_1,count_2,Codon
YAL001C,110,73,AAA
YAL001C,28,22,AAC
...

Each row provides the raw and normalized RFP counts (or two independent count measurements) for a single codon in a single gene, along with the codon identity. Stop codons are excluded. The fasta = FALSE argument to initializeGenomeObject signals that the file contains RFP data rather than coding sequences.

A 500-gene example dataset is included with the package:

rfp_file <- system.file("extdata", "pa_rfpdata.csv", package = "AnaCoDa")

Running the PA model

Step 1: Load genome

library(AnaCoDa)

rfp_file <- system.file("extdata", "pa_rfpdata.csv", package = "AnaCoDa")
genome <- initializeGenomeObject(file = rfp_file, fasta = FALSE)

Step 2: Initialize parameter object

The PA parameter object requires specifying an initial value for the hyperparameter \(s_\phi\) (sphi), which controls the lognormal prior on gene-level expression \(\psi\). With a single gene-set (no mixture) all genes share the same codon-specific parameters \(\alpha\) and \(\lambda'\).

sphi_init    <- 2
numMixtures  <- 1
geneAssignment <- rep(1, length(genome))

parameter <- initializeParameterObject(
  genome          = genome,
  sphi            = sphi_init,
  num.mixtures    = numMixtures,
  gene.assignment = geneAssignment,
  model           = "PA",
  split.serine    = TRUE,
  mixture.definition = "allUnique"
)

Step 3: Initialize model and MCMC objects

model <- initializeModelObject(parameter, "PA")

mcmc <- initializeMCMCObject(
  samples        = 1000,
  thinning       = 10,
  adaptive.width = 50,
  est.expression = TRUE,
  est.csp        = TRUE,
  est.hyper      = TRUE
)

Step 4: Run MCMC

runMCMC modifies mcmc and parameter in place; no return value is needed.

runMCMC(mcmc, genome, model)

Saving and reloading results

Parameter and MCMC objects can be written to disk and reloaded in a later session:

writeParameterObject(parameter, file = "pa_parameter.Rdat")
writeMCMCObject(mcmc, file = "pa_mcmc.Rdat")

parameter <- loadParameterObject(file = "pa_parameter.Rdat")
mcmc      <- loadMCMCObject(file = "pa_mcmc.Rdat")

Diagnostic plots

Log-posterior trace

Inspect convergence by plotting the log-posterior over samples. Discarding an initial burn-in (here 30% of samples) removes the transient at the start of the chain.

logPost <- mcmc$getLogPosteriorTrace()
burnin  <- ceiling(length(logPost) * 0.3)
logPost_post_burnin <- logPost[burnin:length(logPost)]

plot(logPost_post_burnin, type = "l",
     main = paste("log(Posterior), mean =", round(mean(logPost_post_burnin), 1)),
     xlab = "Sample", ylab = "log(Posterior)")
grid(NULL, NULL, lty = 6, col = "cornsilk2")

Hyperparameter traces

trace <- parameter$getTraceObject()
plot(trace, what = "Sphi",             main = "s_phi hyperparameter")
plot(trace, what = "MixtureProbability", main = "Mixture probability")
plot(trace, what = "ExpectedPhi",      main = "Expected expression (psi)")

Codon-specific parameter traces

Trace plots for \(\alpha\) and \(\lambda'\) across all codons help assess per-codon convergence.

plot(trace, what = "Alpha",       mixture = 1)
plot(trace, what = "LambdaPrime", mixture = 1)

Posterior means and credible intervals

After discarding burn-in, extract posterior means and 95% credible intervals for each codon’s \(\alpha\) and \(\lambda'\):

samples    <- 1000
codonList  <- codons()          # 61 sense codons
cat        <- 1                 # mixture category index
burnin_idx <- samples * 0.5    # use posterior half

alpha_mean <- numeric(61)
alpha_ci   <- matrix(0, nrow = 61, ncol = 2)
lp_mean    <- numeric(61)
lp_ci      <- matrix(0, nrow = 61, ncol = 2)

for (i in seq_along(codonList)) {
  codon <- codonList[i]

  alpha_mean[i] <- parameter$getCodonSpecificPosteriorMean(cat, burnin_idx, codon, 0, FALSE)
  alpha_trace   <- trace$getCodonSpecificParameterTraceByMixtureElementForCodon(1, codon, 0, FALSE)
  alpha_ci[i, ] <- quantile(alpha_trace[burnin_idx:samples], probs = c(0.025, 0.975))

  lp_mean[i]  <- parameter$getCodonSpecificPosteriorMean(cat, burnin_idx, codon, 1, FALSE)
  lp_trace    <- trace$getCodonSpecificParameterTraceByMixtureElementForCodon(1, codon, 1, FALSE)
  lp_ci[i, ]  <- quantile(lp_trace[burnin_idx:samples], probs = c(0.025, 0.975))
}

# Plot alpha credible intervals
plot(NULL, NULL,
     xlim = c(1, 61), ylim = range(alpha_ci),
     main = "Posterior mean and 95% CI: Alpha",
     xlab = "Codon", ylab = "Alpha", axes = FALSE)
confidenceInterval.plot(x = 1:61, y = alpha_mean, sd.y = alpha_ci)
axis(2)
axis(1, at = 1:61, labels = codonList, tck = 0.02, las = 2, cex.axis = 0.6)

# Plot lambda-prime credible intervals
plot(NULL, NULL,
     xlim = c(1, 61), ylim = range(lp_ci),
     main = "Posterior mean and 95% CI: LambdaPrime",
     xlab = "Codon", ylab = "LambdaPrime", axes = FALSE)
confidenceInterval.plot(x = 1:61, y = lp_mean, sd.y = lp_ci)
axis(2)
axis(1, at = 1:61, labels = codonList, tck = 0.02, las = 2, cex.axis = 0.6)

Pausing times (in arbitrary units) are proportional to \(\alpha \cdot \lambda'\):

pausing_times <- alpha_mean * lp_mean
barplot(pausing_times, names.arg = codonList,
        main = "Estimated pausing times per codon",
        las = 2, cex.names = 0.6)

PANSE model

PANSE extends PA by jointly estimating per-codon nonsense error (NSE) rates. The model accounts for the fact that ribosomes that drop off upstream reduce the observed RFP counts at downstream positions.

The setup is identical to PA except for the model string:

parameter_nse <- initializeParameterObject(
  genome          = genome,
  sphi            = 2,
  num.mixtures    = 1,
  gene.assignment = rep(1, length(genome)),
  model           = "PANSE",
  split.serine    = TRUE,
  mixture.definition = "allUnique"
)

model_nse <- initializeModelObject(parameter_nse, "PANSE")

mcmc_nse <- initializeMCMCObject(
  samples        = 1000,
  thinning       = 10,
  adaptive.width = 50,
  est.expression = TRUE,
  est.csp        = TRUE,
  est.hyper      = TRUE
)

runMCMC(mcmc_nse, genome, model_nse)

The same diagnostic and posterior extraction functions shown above apply to the PANSE parameter object. NSE rate traces are accessed via what = "NSERates" in plot(trace, ...).

Restart files

For long runs it is useful to save periodic restart checkpoints so that computation can be resumed after interruption:

setRestartSettings(mcmc, file = "pa_restart.rst",
                   every = 50, initialize = TRUE)
runMCMC(mcmc, genome, model)

Pass the restart file to initializeParameterObject via the restart.file argument to resume:

parameter <- initializeParameterObject(model = "PA",
                                        restart.file = "pa_restart.rst")
